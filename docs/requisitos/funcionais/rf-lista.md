### Requisitos Funcionais (RF) - O que o sistema deve fazer

Esta seção lista todas as ações e comportamentos que o sistema deve executar.

| ID       | Descrição                                                                                                                                                                                                                      |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **RF01** | O sistema deve permitir upload de arquivos MP4 e MOV até 50MB.                                                                                                                                                                 |
| **RF02** | O sistema deve armazenar o vídeo original temporariamente no servidor exclusivamente durante a sessão de processamento, descartando-o automaticamente ao término, permitindo a persistência local mediante comando do usuário. |
| **RF03** | O armazenamento local será através do IndexedDB. O 'deck' deverá permitir que funcione mesmo offline, salvando tanto os trechos/'clips' quanto as questões geradas e respostas.                                                |
| **RF04** | O sistema deve processar o vídeo enviado, mapear os tempos de fala e segmentar o arquivo bruto em múltiplos 'clips' sincronizados.                                                                                             |
| **RF05** | O sistema deve agrupar os 'clips' processados em um 'deck' e construir a estrutura de dados das 'lacunas' textuais necessária para o motor de exercícios.                                                                      |
| **RF06** | Os exercícios, antes de começar deverão permitir a escolha dos tipos de exercícios. Variando entre "Múltipla Escolha", "Preencher Lacunas" e "Ditado"                                                                          |
| **RF07** | O sistema deverá entregar o resultado imediato do exercício feito, diferença entre palavras, erros, faltas...                                                                                                                  |
| **RF08** | O sistema deverá possibilitar a consulta da tradução das palavras faladas (caso o usuário clique por isso)                                                                                                                     |
| **RF09** | O sistema deverá possibilitar a consulta da tradução da frase (caso o usuário clique por isso)                                                                                                                                 |
| **RF10** | O sistema deverá entregar um feedback ao usuário passando sobre os erros do exercício comparando o que foi entregue e o que era esperado. Caso haja acertos, basta relatar que não houve erros.                                |
| **RF11** | O player deverá ser similar a outros players de vídeo do mercado, com recursos como Pausa, Volume, Avançar e Voltar.                                                                                                           |
| **RF12** | O player deverá incluir a funcionalidade de alterar a velocidade do vídeo (podendo desacelerar para 0.75x, 0.50x... ) de acordo com o nível de dificuldade selecionado.                                                        |
| **RF13** | O sistema deve oferecer uma variedade de temas, para satisfazer os mais variados gostos e ambientes.                                                                                                                           |
| **RF14** | O sistema deve oferecer alternativas de download que não seja apenas por arquivos brutos `mp4` ou `.mov`.                                                                                                                      |
| **RF15** | No painel dashboard podem ter sugestões de 'clips' para o usuário treinar, conhecer a plataforma e já ter um gatilho.                                                                                                          |
| **RF16** | Um 'deck' com mais de um 'clip' deve entregar eles por partes, ao invés de entregar tudo de uma vez.                                                                                                                           |
