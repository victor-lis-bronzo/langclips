### Requisitos Não Funcionais (RNF) - Como o sistema deve fazer

Esta seção define as restrições técnicas, de segurança, usabilidade e desempenho da plataforma.

| ID        | Descrição                                                                                                                                                                                                                                             | Categoria    | Prioridade |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ | ---------- |
| **RNF01** | O player deve iniciar a reprodução em menos de 2 segundos em conexões de 10Mbps e conseguir sustentar um buffer que evite pausas durante a execução.                                                                                                  | Arquitetura  | Alta       |
| **RNF02** | O cliente web deve suportar degradação de rede implementando retentativas automáticas e limitando o payload de upload a um bitrate máximo predefinido caso a latência da conexão do usuário ultrapasse 500ms.                                         | Média        | Baixa      |
| **RNF03** | A interface deve atingir pontuação mínima de 90 no Lighthouse para Acessibilidade e Performance. Especialmente em aparelhos mobile, como Tablets ou Smartphones.                                                                                      | Layout UI/UX | Baixa      |
| **RNF04** | Em caso de envio de mais de 10 uploads simultâneos (por mais de um usuário), o sistema encaminhará uma fila e o priorizará os uploads por ordem de chegada.                                                                                           | Performance  | Alta       |
| **RNF05** | Deverá ser implementado um rate-limit por IP de no máximo 5 requisições por minuto, para prevenção de ataques de DDoS.                                                                                                                                | Segurança    | Alta       |
| **RNF06** | A interface deve ser construída utilizando os conceitos de "Bento Grid", "Skeleton States" e "Micro-interações" visando uma experiência fluida e moderna para o usuário. Isso aproxima o sistema de layouts de redes sociais.                         | Layout UI/UX | Média      |
| **RNF07** | A interface deve ser totalmente responsiva e funcional em resoluções a partir de 320px de largura                                                                                                                                                     | Layout UI/UX | Média      |
